    JS何时回收一个内存

    满足以下两个条件的《变量》的内存会被回收
    1.该变量是在局部作用域(函数作用域)内声明，而非全局作用域下声明。
    2.没有《任何别域下的变量》可以通过间接或直接的方式访问到它。

    例1：

    function f1(){
    var a = 1;
    var b = 2
    c = 3;
    return function(){
        return a
    }
}
var x = f1();
// 本次f1执行所产生的执行期上下文中有
// a 变量
// b 变量
// 一个匿名函数function

// 在这里f1执行完毕。开始进行垃圾内存检测
// a 可以被 x() 访问到，所以不会被回收
// b没有《任何别域下的变量》可以访问到，会被回收
// 匿名函数function被x访问到。所以不会被回收
// c 相当于在全局window.c上，所以不会被回收

    例2：

    function f1(){
    var a = 1;
    return {
        num: a++
    }
}
var x = f1();
// 本次f1执行所产生的执行期上下文中有
// a 变脸
// 一个对象匿名对象object

// a 可以被 x.num 访问到，所以当f1执行完后不会被回收
// 匿名对象被x访问到。所以不会被回收
