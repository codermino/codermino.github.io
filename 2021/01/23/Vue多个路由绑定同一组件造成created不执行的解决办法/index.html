    Vue多个路由绑定同一组件造成created不执行的解决办法
    2021/01/23 vue Vue多个路由绑定同一组件造成created不执行的解决办法

    出现的问题

    父组件

    <template>
  <div class="father">
    <ul>
      <li @click="toChild(item.id)" v-for="(item,index) in listData" :key="index">
        <span>{{item.id}}</span>
      </li>
    </ul>
  </div>
</template>

<script>
export default {
  name: "Father",
  data() {
    return {
      listData: [
        {id:1,name:'zs'},
        {id:2,name:'ls'},
        {id:3,name:'ww'},
        {id:4,name:'zl'}
      ]
    }
  },
  methods: {
    toChild(id) {
      // this.$router.push({
      //   path: `/child/${id}`,
      // })

      // this.$router.push({
      //   name: 'child2',
      //   params: {
      //     id: id
      //   }
      // })

      this.$router.push({
        path: '/child2',
        query: {
          id: id
        }
      })
    }
  },
}
</script>

    子组件

    <template>
  <div class="child">
    <span>{{id}}</span>
  </div>
</template>

<script>
export default {
  name: "child",
  created() {
    this.id = this.$route.params.id;
  },
  data() {
    return {
      id: ''
    }
  }
}
</script>

    问题描述
    点击不同的li标签之后跳转到child页面，地址栏的地址发生变化
    但是id没有发生改变。因为页面没有刷新，
    也就是说created函数只执行了一次。

    问题原因

    官方解释: https://router.vuejs.org/zh/guide/essentials/dynamic-matching.html[官方解释](https://router.vuejs.org/zh/guide/essentials/dynamic-matching.html)

    比如都访问user组件,传送到参数分别foo和bar,user组件只会使用一次.

    解决方法

    监听$route的变化，然后进行赋值

    <template>
  <div class="child2">
    <span>{{id}}</span>
  </div>
</template>

<script>
export default {
  name: "child2",
  created() {
    // this.id = this.$route.params.id;
    this.id = this.$route.query.id;
  },
  watch:{
    $route:{
      handler(val,oldval){
        this.id = val.query.id
      },
      // 深度观察监听
      deep: true,
      immediate: true
    }
  },
  data() {
    return {
      id: ''
    }
  }
}
</script>
